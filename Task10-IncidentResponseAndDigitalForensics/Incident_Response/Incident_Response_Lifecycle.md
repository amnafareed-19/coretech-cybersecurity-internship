# Incident Response Lifecycle

Incident response is a structured approach used to detect, manage, and recover from cybersecurity incidents.

## 1. Preparation

Preparation involves establishing security policies, creating response procedures, training employees, maintaining backups, and deploying monitoring tools.

### Activities

* Security awareness training
* Backup implementation
* Incident response team formation
* Security monitoring setup

---

## 2. Detection and Analysis

This phase identifies and validates security incidents.

### Activities

* Monitoring alerts
* Reviewing logs
* Malware detection
* Incident classification

---

## 3. Containment

Containment limits the spread and impact of the incident.

### Activities

* Isolating infected devices
* Disabling compromised accounts
* Blocking malicious network traffic

---

## 4. Eradication

The root cause of the incident is removed.

### Activities

* Removing malware
* Deleting malicious files
* Applying security patches

---

## 5. Recovery

Systems are restored to normal operation.

### Activities

* Restoring backups
* Testing restored systems
* Monitoring for reinfection

---

## 6. Lessons Learned

The organization reviews the incident and improves security controls.

### Activities

* Root cause analysis
* Updating procedures
* Improving security awareness
